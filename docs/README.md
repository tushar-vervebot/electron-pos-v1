# docs

**Developer documentation** for the POS system.

## What goes here

| File / Folder | Purpose |
|---|---|
| `architecture.md` | System architecture overview and decision log |
| `plugin-api.md` | Full Plugin API reference for plugin developers |
| `slots.md` | All available UI slots with usage examples |
| `hooks.md` | All available lifecycle hooks with payload shapes |
| `events.md` | All available events with payload shapes |
| `permissions.md` | Permission system reference |
| `theming.md` | CSS variables, theme overrides, and dark mode guide |
| `hardware.md` | Hardware integration guide (printer, barcode, CCTV) |
| `plugin-development-guide.md` | Step-by-step guide to building a new plugin |
| `deployment.md` | Build, package, and distribute the app |
| `diagrams/` | Architecture diagrams (Mermaid or PNG) |

## Existing docs at project root

The following markdown files in the project root also serve as documentation:

- `POS_MODULARITY_AND_FOLDER_STRUCTURE_GUIDE.md` – Architecture guide and folder conventions
- `ARCHITECTURE_DECISIONS.md` – Key architecture decisions and rationale
- `HYBRID_APPROACH.md` – Notes on the hybrid Electron Forge + electron-vite setup
- `AUTO_UPDATE.md` – Auto-update implementation notes
- `OFFLINE_CACHE_AUTH_GUIDE.md` – Offline and cache authentication guide
- `PROJECT_FLOW.md` – End-to-end project flow
- `PROJECT_STRUCTURE.md` – Project structure notes
